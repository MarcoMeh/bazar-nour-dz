1) صفحة تسجيل الدخول (Login Page)

واجهة بسيطة، نظيفة، داكنة (dark) أو فاتحة حسب رغبتك:

• Username
• Password
• زر Login
• تحتها سطر صغير: “اللغة: العربية | Français”

المستخدم الافتراضي:
username: admin
password: astalia
وهذا الوحــــيد اللي يقدر يضيف Users جدد.

====================================================================

2) الواجهة الرئيسية (Main Window)

تتكون من Sidebar على اليسار + Content على اليمين.

Sidebar (ثابت):

أزرار أساسية:

• Dashboard
• قائمة الآثار
• مراجعة قطعة (إن حبيت)
• المستخدمين (Users)
• الإعدادات (Settings)
• النسخ الاحتياطي (Backup) ← (أنا نضيفها لأنها مهمة جداً)
• تسجيل خروج (Logout)

الستايل:
مودرن، أيقونات Material Icons، خلفية رمادية-غامقة، والنص أبيض.

====================================================================

3) Dashboard

واجهة تحليلية فيها:

• عدد الآثار في قاعدة البيانات
• عدد الآثار المسترجعة
• عدد الآثار في حالة سيئة
• عدد الأنواع
• مخطط دائري (Pie Chart) يوضح حالة الحفظ
• مخطط أعمدة (Bar Chart) يوضح الفترات التاريخية
• مخطط آخر لصنف القطع (نوع القطعة)

نستعمل:
QtCharts (متوافق جداً مع PyQt5)

====================================================================

4) صفحة قائمة الآثار (Artifacts List)

جدول حديث (QTableWidget أو QTableView + Model):

الأعمدة:

• رقم القطعة
• اسم القطعة
• نوع القطعة
• العدد
• مادة الصنع
• الفترة
• حالة الحفظ
• زر “عرض”
• زر “تعديل”
• زر “حذف”

أزرار تحت العنوان:

• إضافة قطعة جديدة
• بحث متقدم
• فلترة حسب نوع القطعة / الفترة التاريخية / حالة الحفظ

====================================================================

5) صفحة إضافة / تعديل قطعة أثرية

استمارة فيها:

• اسم القطعة
• نوع القطعة (Combobox يأتي من Settings)
• العدد
• مادة الصنع (Combobox من Settings)
• الفترة التاريخية (Combobox من Settings)
• الوصف (Textarea)
• حالة الحفظ (Combobox)
• طريقة الاسترجاع (Combobox)
• تاريخ الاسترجاع (DateEdit)
• رفع الصور (يدعم عدة صور دفعة واحدة)
• زر حفظ

ID = يولّد تلقائياً ومتصاعد.

====================================================================

6) صفحة تفحص القطعة (Details Page)

تظهر لما تضغط "عرض".

تحتوي:

• كل المعلومات عن القطعة
• صور في Slider (QStackedWidget + Buttons أو QCarousel)
• يمكن تضيف “صور جديدة”
• يمكن تطبع تقرير PDF عن القطعة لو حبيت نضيفها

====================================================================

7) صفحة المستخدمين (Users)

فقط admin يراها.

جدول يحتوي:

• اسم المستخدم
• الدور (Admin / User)
• حالة الحساب
• زر تعديل
• زر حذف

زر فوق: إضافة مستخدم جديد

عند الإضافة:

• username
• password
• role (Admin or User)

====================================================================

8) صفحة الإعدادات (Settings)

هذي المحور الرئيسي لكل الـ Combobox في التطبيق.

قوائم:

• أنواع القطع
• مواد الصنع
• الفترات التاريخية
• حالة الحفظ
• طرق الاسترجاع

كل قائمة عندها:

• إضافة
• تعديل
• حذف
• ترتيب

بهذا الشكل، عند إضافة قطعة أثرية جديدة → Combobox يجيب البيانات أوتوماتيكياً.

====================================================================

9) دعم اللغات (عربية + فرنسية)

نستعمل Qt Linguist أو نظام Localization يدوي:

• ملفات ts/ar
• ملفات ts/fr
• ترجمة كل الواجهات
• زر تغيير اللغة من أي صفحة

نخلي التصميم يدعم RTL (من اليمين لليسار) لما تكون العربية.

====================================================================

10) قاعدة البيانات (SQLite)

أسهل، أسرع، تدعمها كل مكاتب الدولة.

الجداول الأساسية:

• artifacts
• artifact_images
• users
• types (نوع القطعة)
• materials (مادة الصنع)
• periods (الفترة التاريخية)
• conditions (حالة الحفظ)
• retrieval_methods (طريقة الاسترجاع)

====================================================================

11) التصميم (Modern UI)

نستعمل:

• QSS (CSS لـ Qt)
• ألوان مريحة
• أيقونات Fluent Icons أو Material Icons
• Buttons دائرية ناعمة
• Input Forms مرتبة
• Sidebar أنيق جداً

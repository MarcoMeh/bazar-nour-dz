المخطط الفني الشامل للمنصة

هندسة نظام بازارنا (Bazzarna)

المخطط الفني وهيكل العلاقات والبيانات لأصحاب المشاريع والمطورين

1. نظرة عامة على المنصة

منصة بازارنا هي نظام تجارة إلكترونية متعدد المتاجر (Multi-Tenant SaaS E-commerce) موجه خصيصاً للسوق الجزائري. تتيح للملاك إنشاء متاجرهم الخاصة، إدارة منتجاتهم ومخزونهم، استقبال الطلبات، وتجهيز عمليات التوصيل، بينما تتيح للعملاء تصفح المتاجر، مقارنة المنتجات، وإتمام الطلبات عبر السلة أو الواتساب مباشرة بالدينار الجزائري.

تم تصميم هذا النظام ليكون خفيف الوزن وسريع التحميل على شبكات الهواتف المحمولة في الجزائر، ويوفر بيئة تجارة متكاملة تجمع بين سهولة تصفح الزوار وقوة لوحة التحكم الخاصة بالتجار والمسؤولين.

2. البنية التكنولوجية للبرنامج

يعتمد النظام على بنية تحتية سحابية موزعة وخفيفة لتقليل زمن الاستجابة للمستخدمين:

  • الواجهة الأمامية (Front-End): مكتبة React.js مع لغة TypeScript وتجميع Vite لسرعة البناء والتطوير، مستخدمين Tailwind CSS للتصاميم المتجاوبة ونظام Shadcn UI.
  • قاعدة البيانات والـ Back-End: خوادم Supabase المعتمدة على PostgreSQL لضمان موثوقية العلاقات والبيانات الضخمة.
  • إدارة الصلاحيات والولوج (Auth): Supabase Auth لحماية وتشفير حسابات التجار والزبائن والتحكم بالأدوار.
  • تخزين الصور (Storage): Supabase Storage مع مستودعات مخصصة لصور المنتجات وشعارات المحلات.

3. الأدوار وصلاحيات الوصول

الزبون / الزائر

تصفح السلع والشراء

له الصلاحية الكاملة لتصفح المنتجات النشطة للمحلات غير المجمدة والبحث المتقدم بالولايات، وإدارة سلته ومفضلته، وإتمام الطلبات عبر السلة أو الواتساب.

التاجر / الشريك

إدارة المتجر الخاص

له الصلاحية للتحكم وتحديث بيانات محله بالكامل، رفع المنتجات، وإعداد السمات والألوان لمتجره، وتتبع وتجهيز الطلبات وتعديل حالات الشحن.

الأدمن / المسؤول

التحكم السيادي بالنظام

له صلاحية تفعيل المتاجر أو تجميدها يدويًا، مراجعة إحصائيات المبيعات والأرباح، إعداد الفئات العامة وشركات التوصيل وتوزيع الولايات الجغرافي.

5. سير العمليات الحيوية للنظام

أ. قبول المتاجر الجديدة (Merchant Approval):

يقدم التاجر طلبه، وبمجرد موافقة المسؤول في لوحة التحكم، يتم تلقائياً إنشاء الحساب الأمني للتاجر، وتوليد ملف المستخدم (Profile)، وربطه بسجل المحل التجاري مع تفعيل كلمة مرور افتراضية مشتقة من رقم هاتفه.

ب. إتمام الشراء المشترك عبر الـ RPC:

لضمان سرعة وموثوقية المعاملات وتفادي RLS Blockages، تتم كتابة الطلبات وسلعها في عملية ذرية واحدة (Atomic Transaction) في خادم قاعدة البيانات عبر استدعاء الإجراء المخزن create_order، مما يرسل إشعارات فورية للتجار المعنيين.

ج. التحقق من صلاحية عرض المنتجات:

تقوم قاعدة البيانات تلقائياً بفلترة المنتجات ومنع ظهورها للزوار في حال انتهاء اشتراك المتجر التابع له المنتج (تاريخ اشتراك قديم) أو في حال تم تجميد المتجر يدوياً من الإدارة لضمان مصداقية المنصة.

6. التوطين للجمهور الجزائري والسمات

تم مراعاة تكييف المنصة لتعمل كلياً بما يتناسب مع السوق المحترفة للجزائر:

  • الأولوية للغة العربية: استعراض الاسم والوصف العربي للمنتجات والجداول مع وضع الترجمات الأخرى كخيار احتياطي لضمان وصول أكبر للزبائن.
  • دعم 3 سمات تفاعلية للمتجر: يختار التاجر سمة متجره واللون الرئيسي وتنعكس فوراً للزوار:
    • السمة الفاتحة (Light Theme): تصميم كلاسيكي أبيض ناصع ونظيف.
    • السمة الداكنة (Dark Theme): مريح للعين ليلاً وخلفيات بألوان متناسقة.
    • السمة الزجاجية (Glassmorphism): تصميم زجاجي شفاف رائع مع بلورة تصفح جذابة.
